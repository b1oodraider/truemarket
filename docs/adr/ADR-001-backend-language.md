# ADR-001 — Выбор языка бэкенда

- **Статус:** Accepted
- **Дата:** 2026-05-07
- **Автор:** Product Owner (решение), Tech Lead (документирование)

## Контекст

Раздел 4 CLAUDE.md фиксирует выбор между двумя кандидатами:

- **Go** — производительность под нагрузкой, низкое потребление памяти, быстрая сборка, простой деплой;
- **Java 21 + Spring Boot 3** — крупная экосистема, готовые SDK для российских платёжных и compliance-сервисов, большой кадровый рынок в РФ.

Критерии оценки из CLAUDE.md: производительность, скорость разработки, экосистема, стоимость найма команды.

Контекст проекта:
- B2C-маркетплейс с тяжёлой бизнес-логикой (эскроу, расчёт комиссий, маркировка «Честный ЗНАК», 54-ФЗ).
- Значительная доля кода — интеграции с внешними сервисами РФ (ЮKassa, ОФД, маркировка, российские курьеры).
- Целевой рынок — РФ; найм будет вестись на российском рынке труда.

## Решение

**Java 21 + Spring Boot 3.**

Сборка через **Maven Wrapper (`./mvnw`)**. Версия Java фиксируется в `.tool-versions`. Spring Boot 3.3+ (актуальная мажорная LTS-линейка).

## Обоснование

| Критерий | Go | Java 21 + Spring Boot 3 |
|---|---|---|
| Производительность под нагрузкой | ✅ Лучше для CPU-bound и сетевых сервисов | ⚠️ С Virtual Threads (Project Loom, JEP 444) Java 21 закрывает разрыв на I/O-bound нагрузках |
| Скорость разработки бизнес-логики | ⚠️ Меньше готовых решений для финтех/compliance | ✅ Spring Modulith, Spring Data, Spring Security — годы готовых паттернов |
| Экосистема для интеграций РФ | ⚠️ Большинство SDK российских сервисов отсутствуют для Go или поддерживаются комьюнити | ✅ ЮKassa SDK, СБП, маркировка «Честный ЗНАК», ОФД-провайдеры — официальные Java/JVM SDK или Spring-стартеры |
| Стоимость и доступность найма | ⚠️ Меньше специалистов уровня middle/senior на рынке РФ | ✅ Самый широкий пул кадров на Java в РФ |
| Memory footprint | ✅ ~30–80 МБ на сервис | ⚠️ ~250–500 МБ на сервис (нивелируется за счёт `-XX:+UseZGC` и AOT через Spring Native при необходимости) |
| Compile / startup time | ✅ Секунды | ⚠️ 10–30 сек (Spring Boot 3 с context indexer); приемлемо для модульного монолита |

Ключевой довод — **экосистема российских интеграций**. На фазах 2–3 проект массово взаимодействует с ЮKassa (платежи, фискализация), ОФД, «Честный ЗНАК» и российскими курьерами. На Java эти интеграции — день работы; на Go каждая — неделя.

Java 21 с Virtual Threads существенно сокращает разрыв в производительности на I/O-bound сценариях, которые доминируют в маркетплейсе.

## Последствия

### Положительные
- Быстрее MVP за счёт Spring Boot starters и готовых SDK.
- Готовые библиотеки: Spring Security (auth, JWT, OAuth), Spring Data JPA + Flyway (миграции), Spring AMQP (RabbitMQ), Spring WebSocket, Resilience4j.
- Crash-safety: проверенные паттерны для финтеха.
- Большой пул кандидатов на найм.

### Отрицательные / риски
- Выше потребление памяти контейнерами (требует более жирные ноды k8s).
- Дольше старт сервисов — миграция на микросервисы потребует учитывать cold-start.
- Требуется настройка JVM (GC, heap) для прода.

### Что меняется в проекте
1. Структура `backend/` — Maven, не cmd-based:
   ```
   backend/
   ├── pom.xml
   ├── mvnw, mvnw.cmd, .mvn/
   ├── src/main/java/ru/truemarket/
   │   ├── TrueMarketApplication.java
   │   ├── auth/
   │   ├── catalog/
   │   ├── ...
   │   └── common/
   ├── src/main/resources/
   │   ├── application.yaml
   │   ├── application-local.yaml
   │   └── db/migration/   # Flyway
   └── src/test/java/ru/truemarket/...
   ```
2. Команды Make используют Maven-варианты из CLAUDE.md раздел 7.2.
3. Линтер: **Spotless + Google Java Format**, конфиг в `pom.xml`.
4. Интеграционные тесты: **Testcontainers** + JUnit 5.
5. Spring Modulith используется для проверки границ модулей (см. ADR-006).

### Пакетная иерархия
Базовый пакет — `ru.truemarket`. Модули — подпакеты первого уровня. Внутри каждого модуля:

```
ru.truemarket.<module>/
├── api/        # @RestController + DTO
├── domain/    # сущности, value objects, enum, бизнес-инварианты
├── service/   # сервисный слой (use-cases)
├── repository/ # JPA-репозитории
├── external/  # клиенты внешних API (изолированы за интерфейсами)
├── config/    # @Configuration
└── package-info.java   # @ApplicationModule (Spring Modulith)
```

Прямой импорт `ru.truemarket.<module>.domain` или `.repository` из другого модуля **запрещён** — только через `api/` (публичные DTO и интерфейсы сервисов).

## Альтернативы (рассмотрены и отклонены)

- **Go** — отклонён по совокупности: проигрывает в экосистеме РФ-интеграций и доступности найма. Перевешивает выигрыш в startup/memory.
- **Kotlin + Spring Boot** — рассматривался как более выразительная альтернатива Java. Отклонён: меньше кадров на рынке РФ по сравнению с чистой Java, конвенции команды устаканились на Java.
- **Node.js + NestJS** — отклонён: избыточен JavaScript-runtime для финансовых расчётов (precision); меньше промышленных SDK для РФ-compliance.

## Что блокирует

- Команды `make backend-*` (см. Makefile) реализуют Maven-вариант из CLAUDE.md 7.2.
